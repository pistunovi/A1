# A1
Тренувальний репозиторій для пояснення принципів роботи Git  та GitHub
Нехай маємо новий проект з назвою A1, в якому необхідно створити файл за допомогою 
VS Code з іменем hello.py двох версій, 1.0 та 1.1.
Перша версія  файлу складається з однієї команди print(“Hello world”), 
а друга print(“Hello Dolly”). Тобто різниця тільки у словах world та Dolly.
Після їх створення як на комп’ютері так і на GitHub, обидві версії треба 
об’єднати в одну через дії на GitHub. Але ця зміна має відбутися і 
у виділеній для проекту теці на вашому комп’ютері.
